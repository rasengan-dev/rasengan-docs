# PageComponent

`PageComponent` is an abstract class provided by Rasengan.js to define `pages`.

## Usage

<Tabs items={['TypeScript', 'JavaScript']}>
  <Tabs.Tab>
    ```tsx filename="home.page.tsx" showLineNumbers copy /PageComponent/
    import { PageComponent, defineRoutePage } from "rasengan";
    
    class Home extends PageComponent {
      render() {
        return (
          <div>
            <h1>Home Page</h1>
          </div>
        );
      }
    }

    export default defineRoutePage({
      path: "/",
      title: "Home",
      description: "Home page"
    })(Home);
    ```
  </Tabs.Tab>
  <Tabs.Tab>
    ```jsx filename="home.page.jsx" showLineNumbers copy /PageComponent/

    class Home extends PageComponent {
      render() {
        return (
          <div>
            <h1>Home Page</h1>
          </div>
        );
      }
    }

    export default defineRoutePage({
      path: "/",
      title: "Home",
      description: "Home page"
    })(Home);
    ```
  </Tabs.Tab>
</Tabs>

Similar to [`LayoutComponent`](/docs/api-reference/components/layout-component), the `PageComponent` class requires a `render` method to be implemented. 
The `render` method should return a `React.Node` element representing the page.

## Properties and Methods

The `PageComponent` class has the following properties and methods:

### Properties

- `path`: The path of the page. It is used to define the route of the page.
- `title`: The title of the page. It is used to define the title of the page in the browser.
- `description`: The description of the page. It is used to define the description of the page in the browser.

### Methods

- `render()`: This method is used to define the content of the page
- `loader()`: Similar to `getServerSideProps()` in Next.js, this method is used to make some operations on the server before the page is rendered.

The `loader()` method is optional. The returned promise has to follow the following structure:

```ts
type LoaderResponse = {
  props?: { [key: string]: any };
  redirect?: string
};
```

<AlertBox>
- `props`: The props that will be passed to the page component `render` method.
- `redirect`: The path to redirect the user.

The `props` and `redirect` are optional. If the `redirect` is defined, the `props` will be ignored.
</AlertBox>
