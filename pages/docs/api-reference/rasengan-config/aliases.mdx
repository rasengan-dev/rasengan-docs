# Alias imports

Alias imports allow you to shorten and clarify file paths or modules. Instead of lengthy import paths, you can use concise, descriptive aliases.
This not only makes your code cleaner but also more readable.

## Define aliases

<Steps>
### configure your `tsconfig.json` or `jsconfig.json`

This is very useful for autocompletion purposes.

<Tabs items={['TypeScript', 'JavaScript']} defaultIndex="0">
  <Tabs.Tab>
    ```json filename="tsconfig.json" showLineNumbers copy /baseUrl/
    {
      "compilerOptions": {
        "baseUrl": ".",
        "paths": {
          "@components/*": ["src/components/*"],
          "@styles/*": ["src/styles/*"],
          "@app/*": ["src/app/*"]
        }
      }
    }
    ```
    <AlertBox>
      If you are using `tsconfig.json`, make sure to set the `baseUrl` to `.`.
    </AlertBox>
  </Tabs.Tab>
  <Tabs.Tab>
    ```json filename="jsconfig.json" showLineNumbers copy /baseUrl/
    {
      "compilerOptions": {
        "baseUrl": ".",
        "paths": {
          "@components/*": ["src/components/*"],
          "@styles/*": ["src/styles/*"],
          "@app/*": ["src/app/*"]
        }
      }
    }
    ```

    <AlertBox>
      If you are using `jsconfig.json`, make sure to set the `baseUrl` to `.`.
    </AlertBox>

  </Tabs.Tab>
</Tabs>

### Define aliases in `rasengan.config.js`

```javascript filename="rasengan.config.js" copy showLineNumbers
import { defineConfig } from "rasengan";

export default defineConfig({
  vite: {
    resolve: {
      alias: [
        {
          find: "@components",
          replacement: "./src/components",
        },
        {
          find: "@styles",
          replacement: "./src/styles",
        },
        {
          find: "@app",
          replacement: "./src/app",
        },
      ],
    },
  },
});
```

</Steps>
